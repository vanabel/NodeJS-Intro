第三节. Eslint的安装

首先， 我们假设我们所有的NodeJS文件都在src目录下。并已经切换到该目录。此时我们可以通过如下命令来全局安装eslint

npm install -g eslint

注意，这里必须使用全局安装，否则Syntastic插件找不到eslint.

接下来， 我们需要初始化.eslintrc, 在src下新建.eslintrc, 内容如下:

{
  "env": {
    "node": true
  },
  "rules": {
    "array-bracket-spacing": [2, "never"],
    "block-scoped-var": 2,
    "brace-style": [2, "1tbs"],
    "camelcase": 1,
    "computed-property-spacing": [2, "never"],
    "curly": 2,
    "eol-last": 2,
    "eqeqeq": [2, "smart"],
    "max-depth": [1, 3],
    "max-len": [1, 80],
    "max-statements": [1, 15],
    "new-cap": 1,
    "no-extend-native": 2,
    "no-mixed-spaces-and-tabs": 2,
    "no-trailing-spaces": 2,
    "no-unused-vars": 1,
    "no-use-before-define": [2, "nofunc"],
    "object-curly-spacing": [2, "never"],
    "quotes": [2, "single", "avoid-escape"],
    "semi": [2, "always"],
    "keyword-spacing": [2, {"before": true, "after": true}],
    "space-unary-ops": 2
  }
}

测试

作为测试， 你可以新建如下test.js

var HttpTEST=null

然后运行eslint test.js.

test.js|1 col 5 warning| 'HttpTEST' is assigned a value but never used. (no-unused-vars)

表示HttpTest是一个没有使用的变量（浪费资源）。
