第五节. 非阻塞操作

到此为止，似乎一切都运行良好。那么我们继续来处理路由。这次，我们要将/路径得到一个简易编辑器。首先，我们的网页文件如下：

我们的目标很简单，那就是在访问http://localhost:8000/ 时，返回该html文件。因此，我们首先将routeHandler改写如下:

// file: routeHander
var fs=require('fs');
var html='The html content.';

function get(){
    console.log('Get handler is called.');
    fs.readFile(__dirname + '/index.html', function(err, html){
        if (err)  {
            throw err;
        }
        html = html.toString();
        console.log(html);
    });
    return html;
}

function post(){
    console.log('Post handler is called.');
    return 'Hello, world!';
}

exports.get = get;
exports.post = post;

上面的代码比较明显，我们首先载入了Node.js的文件处理模块fs，然后用它将我们建立好的index.html文件读出来放到变量html中。然后返回给get函数， 作为路由处理的结果。试运行下，看看能否如你预期？

运行之后，我们发现控制台正确输出了文件内容。但是网页显示的却只有

The html content.

导致上述问题的原因就在于，Node.js采用的是非阻塞操作，即读取文件这个操作中我们用了一个匿名函数来回调读取结果，程序在读取完成后才返回给回调函数，它并不等待这个操作完成，而是继续执行后续的return html; 这时，html的值当然还没有改变（我们还没读完文件）。

你可能会说这个非阻塞操作真不好，但是换一个角度你就知道这好处很多。试想如果读取一个很大的文件，我们势必要非比较久的时间（所谓的耗时操作，试想一个非常复杂的计算），那么我们的网页这时如果是等操作完成，那么就会卡在这步，对后面的请求延长响应，这对用户是非常不友好的。

下面，我们来看如何修复这里碰到的问题。

阻塞方式

一种简单但不好的方式是采用阻塞方式，即利用Node.js提供的同步读取函数，等文件读取完成后才进行下一步操作。这只需修改原来的fs.readFile为fs.readFileSync. 

// file: routeHander
var fs = require('fs');
var html = 'The html content.';

function get(){
    console.log('Get handler is called.');
    html = fs.readFileSync(__dirname + '/index.html');
    html = html.toString();
    console.log(html);
    return html;
}

function post(){
    console.log('Post handler is called.');
    return 'Hello, world!';
}

exports.get = get;
exports.post = post;

注意，此时由于是同步读取，故readFileSync没有回调函数，详细参考Node.js文档。

重新启动node index.js，打开浏览器刷新发现我们需要的简易编辑器出来了。尽管这次浏览器反应很快（毕竟文件很小), 但是我们已经讲过阻塞操作的坏处多多。故下面来看非阻塞的方式。

非阻塞方式

那么怎么用非阻塞方式得到我们想要的结果呢？回忆非阻塞读取文件readFile采用了回调的方式将结果返回。我们需要将这个结果返回给服务器的res.write()，故我们直接将res传递进来，并接受readFile的回调结果。首先改写我们的get()函数如下：

// file: routeHander
var fs=require('fs');
var html = 'The html content.';
function get(res){
    console.log('Get handler is called.');
    res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
    fs.readFile(__dirname + '/index.html', function(err, data){
        if (err)  {
            throw err;
        }
        html = data.toString();
        console.log(html);
        res.write(html);
        res.end();
    });
}

function post(res){
    console.log('Post handler is called.');
    res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
    res.write('Hello, world!');
    res.end();
}

exports.get = get;
exports.post = post;

即这里，get和post都接收一个参数res，利用它我们可以直接将服务器的响应写在routeHandler模块中。那么这个res是怎么由server传过来的呢？其实，是server传给路由router，然后再由路由router传给routeHandler. 我们首先来看server的改写:

// file: server.js
var http=require('http');
var url=require('url');
function init(handler, router){
    function onRequest(req, res){
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');
        router(handler, path, res);
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

这里我们做了两件事情：

  • 改写router函数，使它可以将服务器的响应绑定到router;
  • 移除原来的http头文件响应，因为这部分我们已经放到上面的routeHandler

接下来，我们还需要改写router, 使它绑定了服务器响应res:

// file: router.js

function route(handle, path, res){
    console.log('Handle request from: ' + path);
    if (typeof handle[path] === 'function'){
        return handle[path](res);
    } else {
        console.log('404 not found');
        res.writeHead(404, {'content-type': '404 not found'});
        res.end();
    }
}

exports.route = route;

这里，我们做了两件事情：

  • 改写router函数以及handle函数，使它们传递res；
  • 重新处理了路径找不到的情况

自此，改造已经完成。我们有了一个非阻塞的简易编辑器网页。具体如何排版是我们接下来的内容。
