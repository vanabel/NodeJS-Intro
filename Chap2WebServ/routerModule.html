第三节. 路由模块

第三节. 路由模块

我们首先来看如何将当前的index.js分离成服务器模块server.js.

自定义模块---server

我们的server.js模块如下

// file: server.js
var http=require('http');
function init(){
    function onRequest(req, res){
        console.log('Request received');
        res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
        res.write('世界，你好!');
        res.end();
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

调用上述模块，我们只需在index.js中如下写即可

// file: index.js
var server=require('./server');

server.init();

我们总结下， 模块其实和普通的js文件没有多大区别，只是我们需要用

exports.exportFun = innerFun

来将内部函数innerfun暴露给外部, 这样外部文件可以通过

var moduleName = require('./moduleFile')

moduleName.exportFun()

来调用了。

理解了过后，我们来看如何写路由模块。

路由模块的实现

仿照前面建立server.js的方法，我们首先建立如下的路由模块，它接收一个参数，即当前请求的路径。

// file: router.js

function route(path){
    console.log('Handle request from: ' + path);
}

exports.route = route;

那么如何使用呢？自然地，我们应该首先从onRequest的req参数中提取path, 然后在回调函数onRequest中处理该路由。例如， 对path为/的请求，我们和以前一样， 返回世界你好。

强耦合方式

我们首先来看一种简单直接的方法。下面的代码，我们用到了自带的url模块，它有一个parse方法， 可以将req.url这个请求地址分离得到pathname.

// file: server.js
var http=require('http');
var url=require('url');
var router=require('./router');
function init(){
    function onRequest(req, res){
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');
        router.route(path);
        res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
        res.write('世界，你好!');
        res.end();
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

运行后得到

Request from / received
Handle request from: /

很有可能，你还会收到/favicon.ico，它是浏览器读取网站的图标（Safari的话，编辑苹果官方主页的地址栏，你会看到一个苹果图标), 这解释了前一节的两次请求。

上面的方法有什么问题呢？

弱耦合方式---依赖注入

上述硬编码方式存在以下问题：

  • server模块中用到了router模块，这样子单元测试时，由于模块的依赖时硬编码的，router的缺陷可能会影响对server模块对测试。这样就很难从单元测试的结果中追踪代码错误。
  • router不可配置。试想我们只有一个router, 那么当不断增加功能时，我们的路由可能变得越来越大，从而变得难以扩展。这样，我们一般是每个组件对应一个路由，好处是每个组件模块都是自包含的，用的时候不需要查找对应的路由。
  • router不可复用. 如果在其他模块中，我们需要调用该router那么我们势必要重新写一遍上面的硬编码。注入依赖相当于为此提供了一个接口，不依赖于router的具体实现。
  • 构建和使用router模块不是分离的。上述代码中我们首先载入了router模块，然后在server模块中使用。注入依赖将使得这个过程分离，从而使得代码更加干净。

下面具体来看怎么依赖注入：

// file: server.js
var http=require('http');
var url=require('url');
function init(router){
    function onRequest(req, res){
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');
        router(path);
        res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
        res.write('世界，你好!');
        res.end();
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

这里就指明了server模块的init组件是依赖于router的，至于router如何实现它不管。此外我们也移除了构建router的代码，因为我们将其直接注入到了init中。注意到server只是一个模块，故我们还需将真实的router及其方法注入到server模块中，即

// file: index.js
var server=require('./server');
var router=require('./router');
server.init(router.route);

运行后看到和前面一样的结果。接下来我们将看到如何处理路由。
