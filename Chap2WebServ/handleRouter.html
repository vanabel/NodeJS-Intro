第四节. 路由功能的实现

第四节. 路由功能的实现

首先，我们知道，路由的基本功能就是针对请求不同的路径，给出不同的处理。也即这里有一个对应，路径->处理请求。此外，我们还应该保持分离性的原则，即路由本身是不管如何处理的，它只是告诉服务器怎么去找到处理函数。

下面我们首先来构建routeHandler模块

// file: routeHander
function get(){
    console.log('Get handler is called.');
}

function post(){
    console.log('Post handler is called.');
}

exports.get = get;
exports.post = post;

上面我们定义了两种最常见的请求，一个是get, 一个是post.

和以前一样，我们如何将这个模块注入到路由中呢？首先我们尝试将router改造如下

// file: router.js
var handler = require('./routeHandler');
var handle = {};
handle['/'] = handler.get;
handle['/post'] = handler.post;

function route(path){
    console.log('Handle request from: ' + path);
    handle[path]();
}

exports.route = route;

运行后可以正常输出log如下：

server started at http://localhost:8000
Request from / received
Handle request from: /
Get handler is called.

依赖注入路由处理函数

明显的问题是，上面我们是将routeHandler硬编码到router模块的。此外，我们还将路由的分类也放到了路由中，这其实不归路由管，路由只管给我路径，我告诉你怎么处理。至于你具体怎么处理以及如何对应我不管。具体怎么处理应该是routeHander的职责，而如何对应应该是app(这里即index.js应该告诉我的。故和前面一样，我们先将其改造为依赖注入的方式

// file: router.js

function route(handle, path){
    console.log('Handle request from: ' + path);
    handle[path]();
}

exports.route = route;

这里，我们申明了依赖关系: router依赖于handle. 接下来当然是在主文件index.js中注入handle, 不难想到应该这样写

// file: index.js
var server=require('./server');
var router=require('./router');
var handler=require('./routeHandler');

var handle = {};
handle['/'] = handler.get;
handle['/post'] = handler.post;

server.init(handle, router.route);

等等，这里我们观察到，我们还需更改server.js中init函数，使得它将handle真正的传给router, 这样我们前面改造的router才知道如何handle。

// file: server.js
var http=require('http');
var url=require('url');
function init(handler, router){
    function onRequest(req, res){
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');
        router(handler, path);
        res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
        res.write('世界，你好!');
        res.end();
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

运行上述改造后的代码，和原来硬编码的方式的结果一样，只不过这次我们的路由只管路由，非常干净。主文件index.js也非常好了，它定义了具体的路径应该怎么处理。剩下不好的地方是server模块，作为服务器，响应请求无可厚非，但返回世界， 你好！应该是路由处理模块的处理结果才对啊！

服务器模块的净化

如同前面讲到的，我们应该将服务器模块中具体内容的响应部分交给routeHandler.

// file: server.js
var http=require('http');
var url=require('url');
function init(handler, router){
    function onRequest(req, res){
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');
        res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
        var resContent = router(handler, path);
        res.write(resContent);
        res.end();
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

当然， 我们需要改造routeHandler, 使得它返回世界，你好！

// file: routeHander
function get(){
    console.log('Get handler is called.');
    return '世界，你好!';
}

function post(){
    console.log('Post handler is called.');
    return 'Hello, world!';
}

exports.get = get;
exports.post = post;

至此，你是不是和我一样觉得可以了呢？运行代码试试，发现如下错误:

TypeError: First argument must be a string or Buffer
at write_ (_http_outgoing.js:645:11)
at ServerResponse.write (_http_outgoing.js:620:10)
at Server.onRequest (~/devlopment/gitbooks/nodejs-intro/src/server.js:10:7)

上面错误的意思是在server.js的10行7列, 我们用到了write函数，这个函数第一个参数要求是字符串或者流。你给我的不是这种类型的，故我不知道怎么处理了！

回忆，这个变量是由router处理的结果定义的。回头看看router， 它处理的结果并没有返回， 故我们需要将其中的handle[path]();改为return handle[path]();即可。事实上，原来默认的返回是undefined, 因为我们并没有返回。这可以用res.write(typeof resContent)得到。利用这个思想，我们还可以定义默认路由，即那些无法处理的路由（别忘了试试http://localhost:8000/post, 并观察控制台情况！将post改为get如何？）

更加健壮的路由模块

通过实验发现，访问http://localhost:8000/get 服务器停止响应了。这是因为我们并没有定义/get这个路径应该怎么去处理。这时会报告handle[path]不是一个函数。为了修复这个问题，使得我们的服务器变得更加强壮，我们通过一个type判断来修复如下

// file: router.js

function route(handle, path){
    console.log('Handle request from: ' + path);
    if (typeof handle[path] === 'function'){
        return handle[path]();
    } else {
        return '404 not found';
    }
}

exports.route = route;
