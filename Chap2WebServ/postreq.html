第七节. 使用POST发送请求

第七节. 使用POST发送请求

尽管上面我们已经基本实现了markdown编辑器的基本功能。这里作为示例，我将演示如何利用NodeJs处理Post请求。事实上，我们将看到如何把前面的showdown.js脚本作为后台来使用。此外，我们也可以学习NodeJS如何处理POST请求。

POST请求的发送

首先，我们将改写我们的usr.js脚本，使得它可以实时的发送POST请求到服务器，然后服务器将处理POST请求并返回经过showdown.js处理的marddown文本。

function update(){
    var text = document.getElementById('editor');
    var oldText = text.value;
    var preview = document.getElementById('preview');
    var timeout = null;

    function handleChange(){
        var newText=text.value;
        if (newText === oldText) {
            return;
        } else {
            oldText=newText;
        }
        var xhr = new XMLHttpRequest();
        xhr.open('POST', '/', true);
        xhr.setRequestHeader('Content-Type', 'application/json');
        xhr.send(newText);
        xhr.onreadystatechange = processRequest;
        function processRequest(){
            if (xhr.readyState === 4 && xhr.status === 200 ){
                var req = xhr.responseText;
                preview.innerHTML = req;
            }
        }
    }

    function handle(){
        if (timeout) {
            clearTimeout(timeout);
        }
        timeout=setTimeout(handleChange, 50); }

    text.onkeydown=text.onkeyup=text.onclick=handle;
}

update();
document.getElementById('editor').focus();

你可以看到，我们移除了showdown.js处理的部分， 并且将handleChange函数改为了发送POST请求并接受请求的返回值。这里我没有采用高级的jQuery/ajax方法，因为我想尽可能的保持代码的简洁性。关于XMLHttpRequest的更多资料不难在网上找到。这里的要点是我们使用open/send打开了一个POST请求并将我们的新内容newText发送到服务器，然后通过onreadystatechange判断请求是否发送完毕并返回响应，我们的响应处理函数processRequest只是单纯的将响应的返回写到预览的内置html， 这是因为我们的数据其实是在服务器后台处理的。

也许你会奇怪， 我们发送请求的路径是/, 即当前页面，这主要是因为我不想通过header的Location重定向。那么问题是：

  • 服务器怎么接收POST的数据？
  • 路由怎么区分POST/GET请求呢？
  • 怎么处理POST数据？

我们上面已经说到，为了处理POST请求，我们首先需要服务器接收POST过来的数据，然后将其传给路由，再由路由处理器处理后反馈给该请求。这样， 我们的usr.js就可以接收到该反馈并将其写入到预览的框框中。

下面我们将逐条来看。

服务器模块的改写

具体地，由于POST请求一般是比较"重"的请求，即比较消耗时间与服务器资源。一种比较好的做法是将其拆成比较小的block来发送。这在NodeJS中是用监听器来实现的。

// file: server.js

var http=require('http');
var url=require('url');
function init(handler, router){
    function onRequest(req, res){
        var postData = '';
        var path = url.parse(req.url).pathname;
        console.log('Request from ' + path + ' received');

        if (req.method === 'POST'){
            req.setEncoding('utf8');
            req.addListener('data', function(postDataChunk){
                postData += postDataChunk;
                console.log('Received POST data chunk: '
                    + postDataChunk + '.\n');
                if ( postData.length > 1e6 ){
                    request.connection.destroy();
                }
            });
            req.addListener('end', function(){
                path += 'post';
                router(handler, path, res, postData);
            });
        } else {
            router(handler, path, res);
        }
    }
    http.createServer(onRequest).listen(8000);
    console.log('server started at http://localhost:8000');
}

exports.init = init;

可以看到，相比于以前，我们设置了一个判断req.method是不是POST，如果不是，那么我们直接用以前的路由即可；否则我们就要特别地处理。处理POST请求主要是接收POST传进来的数据，这里我们看到了两个事件监听器：data/end, 那么分别表示有数据传进来/数据接收完毕。在接收数据的时候，我们还设置了一个数据大小的检验（防止有人传大数据来黑服务器），即大小不能超过一百万byte，大约是1M. 当接收完毕后，我们就将完整的数据传递给路由。 可以看到，路由函数多了个参数，故接下来我们来看路由模块的改写。

路由模块的改写

细心的你也许发现了，我们在上面的服务器模块中，将POST路由的路径改写成了/post, 还记得我们原来的请求路径吗？对的， 它是/. 而为了处理该路由，我们的路由映射是

// file: index.js
handle['/post'] = handler.post;

这就是我们改变了路由地址的原因。一方面我们可以区别GET/POST请求；另一方面由于我们的GET/POST请求都是同一个页面，故我们需要这个区分。

我们已经准备好了如何路由POST请求了。

// file: router.js

var path = require('path');

function route(handle, fpath, res, postdata){
    var hpath = fpath,
        ext = path.extname(hpath),
        extarr = ['.js', '.map', '.css'];
    if ( extarr.indexOf(ext)>-1 ) {
        hpath = '/getfile';
    }
    console.log('Handle request from: ' + fpath);
    if (typeof handle[hpath] === 'function' ){
        if (hpath === '/post'){
            return handle[hpath](fpath, res, postdata);
        } else {
            return handle[hpath](fpath, res);
        }
    } else {
        console.log('404 not found');
        res.writeHead(404, {'content-type': '404 not found'});
        res.end();
    }
}

exports.route = route;

可以看到，关键的不同就是我们通过判断路由的路径是不是/post来讲路由传递给不同的路由处理函数，如果是/post， 那么我们就将由服务器传进来的数据postdata经过路由传递给路由处理函数。

路由处理模块的改写

由于我们的路由处理函数只是对POST请求不同，而处理POST请求对路由处理函数是handler.post, 故关于GET部分是完全一样的，下面的代码我省略了这部分。

其实这个POST响应时很简单的，如果不加上showdown.js处理部分，我相信你能过看懂。而showdown.js的处理和我们一起没有多大区别。只是这次我们通过require载入了showdown模块。

    converter = new showdown.Converter();
var html = 'The html content.';

    // the same

function post(fpath, res, data){

    //console.log(data);
    res.writeHead(200, {'content-type': 'text/html; charset=utf-8'});
